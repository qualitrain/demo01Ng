Inyección de Dependencias: Componente 
consulta-personas
(dos instancias) y servicio 
ProvDatPersonas
Ejemplo de implementación de un servicio que es inyectado en las instancias de un componente .
  • Cada instancia del componente de consulta solicita, con un constructor que recibe un objeto del mismo tipo que el del servicio solicitado, la inyección de un servicio de datos (un arreglo de personas).
  • Hay un servicio ProvDatPersonaService que es instanciado por el inyector de dependencias de Angular
  • El servicio está decorado con @Injectable({ providedIn: 'root'}), que indica que debe haber una instacia por aplicación.
  • El componente puede configurar la creación de una instancia por cada instancia del componente agregando a su decoración @Component, un campo providers como el siguiente: providers: [ ProvDatPersonaService ].
  • El servicio puede servir como una memoria compartida entre distintas instancias de uno o más componentes.

Función 
bindCallback
: Componente 
test-rxjs03
  • Este ejemplo muestra como envolver una función asíncrona dentro de otra que devuelve observables
  • Con el uso de bindCallback se envuelve, en un proxy, una función asíncrona que devuelve un número aleatorio.
  • Se invoca a la función por medio del proxy, que devuelve observables
  • Con cada click se crea una nueva suscripción. En ella se provee el callback que espera la función asíncrona.
  • El callback agrega una insignia a la vista, conteniendo los resultados devueltos por cada invocación.
  • De acuerdo al valor del número al momento del click es el tiempo de espera de cada invocación. Por ello, los resultados salenen un orden no secuencial en la mayoría de los casos.
Pipeline en RxJs : Componente 
test-rxjs04
Uso de método pipe y de operadores de mapeo y filtrado sobre eventos
  • Este ejemplo muestra como usar los operadores de RxJs.
  • Se crea un observable sobre los clicks hechos en un <div> que a su vez contiene un elemento <span> por cada mes.
  • Se crea una tubería de procesamiento (pipe) a partir del observable
  • La tubería:
    1. Extrae el elemento html del evento con el operador map
    2. Filtra los elementos span con el operador filter
    3. Extrae de los spans el texto (que contiene el nombre de un mes), con el operador map
    4. Audita el mes con el operador tap
    5. Filtra los meses que correspondan con temporada baja, con el operador filter
    6. Genera un observable con el resultado final de la tubería
  • La suscripción sobre el observable anterior, actualiza el estado para que se despliegue un mensaje que informa sobre promociones en temporada baja.
